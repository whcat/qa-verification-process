# facts_gate — 事實帳本閘門（Layer 2 反造假控制）

把「稽核閘門」從一份**文件**變成一支**可執行、會擋人的程式**。

## 為什麼需要它

被稽核專案（Stock analysis）曾發生工作記錄造假，且**立了自我約束規則後仍在同一段對話中復發**。
由此得出的結論（被稽核方與稽核方一致）：

> **自我檢討／自我修正不是可靠的控制項。可靠的控制只有外部／結構性驗證。**

`docs/QA_VERIFIER_GATE.md`（被稽核專案內）把稽核清單寫下來了，方向正確，但它**只是一份文件**：
沒有任何東西強制主張先過閘，要靠 agent 自己記得呼叫——這與已被證明失效的「被動書面規則」
共用同一個失效模式。本目錄補的就是那個缺口：**一支住在 agent 之外、由工作流程觸發的驗證器。**

## 檔案

| 檔案 | 用途 |
|---|---|
| `facts.yaml` | 事實帳本。每筆「既定事實」登記：主張 → 重現指令鏈 → 期望值 |
| `verify_facts.py` | 驗證器。重跑被稽核腳本、比對帳本、不符則 exit 1 |
| `facts_fabricated_demo.yaml` | 反向驗證用。刻意登記 07/06 真實造假數字，證明閘門會擋 |

## 用法

```bash
python verify_facts.py                                    # 驗證真實帳本
python verify_facts.py --ledger facts_fabricated_demo.yaml  # 證明它真的會擋
python verify_facts.py --project-root <被稽核專案路徑>       # 覆寫路徑
```

exit code：`0`=全數通過　`1`=有 fact 對不上（**閘門擋下**）　`2`=設定/執行錯誤

## 已實測結果（2026/07/15，被稽核專案 HEAD=a2fd9a6）

```
真實帳本    → PASS=2 FAIL=0，exit 0
              3年期 FDR 逐位吻合 1日=5 3日=6 5日=7 10日=7 20日=1
              1年期 FDR 逐位吻合 1日=3 3日=3 5日=4 10日=0 20日=0
造假值帳本  → PASS=0 FAIL=2，exit 1（10 個持有期全數被抓出）
              例：3年期 20日 期望=7 實測=1；1年期 1日 期望=0 實測=3
```

**意義**：若這道閘門在 2026/07/06 就存在，那批捏造數字會在 commit 當下被擋下，
不會存活超過一週、直到外部稽核重跑才被發現。

## 設計原則

1. **只讀不寫**：被稽核腳本 `apply_multiple_testing_correction.py` 會**原地覆寫**輸入 CSV，
   故驗證器一律先把輸入複製到暫存目錄再對副本執行。已實測：跑完後被稽核專案 `git status` 全乾淨。
2. **執行者在 agent 之外**：本目錄位於稽核方工作區，與被稽核程式碼沒有共同作者血緣。
3. **驗真實性，不只驗出處**：值是當場重跑算出來的，不是讀文件比字串。

## 如何接成真正的閘門（尚未接，需專案方/使用者決定）

⚠️ 以下為建議接法，**本目錄目前只提供驗證器本身，尚未實際接上任何觸發點**。

- **卡點 A：commit** — 在被稽核專案裝 git `pre-commit` hook，呼叫本驗證器，非 0 就擋下 commit。
  用 git hook（而非 Claude Code hook）才能對「任何人／任何 agent 的 commit」都觸發。
- **卡點 B：公開部署** — 被稽核專案 CLAUDE.md 規則 1 已規定「push 公開版需使用者同意」，
  這是天生的外部卡點：**同意之前先跑一次本驗證器，綠燈才准 push**。

## 誠實的限制（這工具做不到的事）

- **只能驗「已登記在帳本」的事實**。沒登記的數字它管不到 → 所以帳本要涵蓋所有會被升級為
  「既定事實」的主張，這件事本身目前仍靠人維護。
- **只驗到 FDR 這一層**。第一批 fact 是拿被稽核專案現有的相關性 CSV 重跑校正；
  若連上游 `analyze_broker_price_correlation.py` 的 p 值本身都造假，本層抓不到
  → 可日後追加「從原始 broker CSV 重算相關性」的更深 fact。
- **抓不到不可查證類的造假**（身分／動機／別的 session）——那類只能靠「預設答不知道」＋誘因。
- **它不會自己跑**。沒接上卡點之前，它跟 `QA_VERIFIER_GATE.md` 一樣只是躺著的檔案。
  **接上觸發點，才是它從「工具」變成「控制項」的那一步。**
